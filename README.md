# Telegram to Supabase Importer (Unified)

Проект для автоматического сбора, классификации и структурирования событий из Telegram-каналов с использованием **Google Gemini 2.0 Flash Lite**, **OpenRouter** или **Ollama** (локально) и Supabase.

## 🚀 Deployment (VPS)
Проект развернут на сервере `server.reloto.ru` (148.230.107.136).

### Расположение
- **Путь:** `/root/scripts/bao_tg_importer`
- **Окружение:** `venv` (Python 3.12)
- **Логи крона:** `/root/scripts/bao_tg_importer/logs/cron.log`
- **Лог импорта:** `/root/scripts/bao_tg_importer/log.md` (синхронизируется при пулле)

### Расписание (Cron)
Скрипт запускается автоматически **каждый час с 07:00 до 23:00** (по времени Буэнос-Айреса / GMT-3).
- В UTC (время сервера) это: `10:00 - 02:00`.

### 🔄 Как обновлять (Deploy)
Для внесения изменений (в код или промпты):
1. Отредактируйте файлы локально.
2. Запустите скрипт деплоя в корне проекта:
   ```bash
   ./deploy.sh (в CLI скажи "деплой")
   ```

**Схема деплоя:**
1. **Local → GitHub:** Скрипт делает коммит и пуш (`git push`) в репозиторий.
2. **SSH Trigger:** Подключается к VPS и дает команду на обновление.
3. **VPS → GitHub:** Сервер скачивает свежий код (`git fetch && git reset --hard origin/master`) и обновляет зависимости (`pip install`).

### 🔔 Мониторинг и Оповещения
На сервере настроена система уведомлений о сбоях через Telegram.
- **Механизм:** Wrapper-скрипт `run_cron_importer.sh` следит за кодом выхода Python-скрипта.
- **Триггер:** Если импортер завершается с ошибкой (exit code ≠ 0).
- **Действие:** Отправка сообщения через бота **@baoShopTest_bot**.
- **Получатель:** Admin (ID: 159194550).
- **Логи:** Детали ошибки можно найти в `/root/scripts/bao_tg_importer/logs/cron.log`.

## Основные возможности
- **Unified Prompt:** Все скрипты используют единую инструкцию для ИИ (`!Промты/unified_ollama_prompt.md`).
- **Multi-LLM Support:** Поддержка Google Gemini, OpenRouter (включая бесплатные модели) и локальной Ollama.
- **Multi-Event Support:** Если в одном сообщении Telegram указано несколько дат (до 3-х), система создаст отдельную запись в базе данных для каждого события.
- **Автоматическая загрузка фото:** Изображения из постов загружаются в Supabase Storage (бакет `events`).
- **Интеллектуальная обработка:** Извлечение названия, описания, даты, времени, места, цены и категории.
- **Автоматическое определение channel_id:** Автоматическое обновление `channel_id` в базе данных, если он не указан, на основе `channel_name`.
- **Поддержка топиков:** Поддержка обработки сообщений из отдельных топиков (ветвей обсуждений) в каналах-форумах.
- **Дедипликация:** Автоматическое предотвращение дубликатов событий при импорте.

## Структура проекта
- `scripts/unified_importer.py` — основной импортер с расширенным логированием и поддержкой Gemini. Поддерживает обработку каналов и топиков, автоматически обновляет channel_id.
- `scripts/ollama_supa_json.py` — скрипт для постобработки (заполняет пустые поля в существующих записях).
- `!Промты/unified_ollama_prompt.md` — **главный файл инструкций для AI**.
- `.env` — конфигурация (API ключи, URL базы). **Не хранится в Git!**
- `deploy.sh` — скрипт для автоматического деплоя на VPS.

## Настройка промптов
Все изменения в логике работы ИИ должны вноситься в файл `!Промты/unified_ollama_prompt.md`. 
**Важно:** 
- Промпт настроен на возврат **чистого JSON**.
- Год по умолчанию — **2026**.
- Поле `is_event` определяет, будет ли сообщение сохранено.
- Относительные даты ("сегодня", "пятница") рассчитываются от даты поста.

## Настройка каналов (channel_sync_state)
Для каждого канала в таблице `channel_sync_state` крайне важно правильно заполнять поля:

- **channel_id**: Уникальный числовой идентификатор. 
    - **Важно:** Для каналов и супергрупп он ОБЯЗАТЕЛЬНО должен начинаться с `-100` (например, `-1001699038315`).
    - Использование числового ID вместо `@username` делает работу скрипта намного стабильнее (защита от смены имени канала).
    - **Автоматическое обновление:** Если `channel_id` не указан, скрипт попытается автоматически определить его по `channel_name`.
- **channel_name**: Юзернейм канала с символом `@`. Используется как запасной вариант (fallback), если ID не указан или не найден.
- **thread_id**: ID конкретной темы (топика), если канал является форумом. Оставьте пустым, если требуется импортировать все сообщения канала.

## Как узнать ID канала
В папке `scripts/` подготовлен специальный скрипт `get_channel_id.py`. 
Просто запустите его и введите юзернейм канала:
```bash
source venv/bin/activate
python3 scripts/get_channel_id.py
```
